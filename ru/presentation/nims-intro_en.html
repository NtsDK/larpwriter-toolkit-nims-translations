Concept

During making LARP game story master defines character goals, values and motivations. If story master give instructions like "you are stupid, you hate king, you want to merry princess" then players will ask many questions or just ignore instructions. Thats why player receive not instructions but stories where his character participated. These stories gives information about pregame events, character behavior, estimations, thoughts etc. Also each story looks different for each story character because no one has full information and has subjective story vision. This approach is effective but it is rather difficult.

To solve this task we created NIMS - story master toolkit. In this presentation we will tell you how we describe LARP story making process with NIMS.

Terminology

  • Character – LARP game character.

  • Story – series of connected events which involves some number of characters.

  • Master story – story master version of how stories actually happen.

  • Story adaptation – character subjective vision of story events.

  • Handout – player document with character information, story adaptations and some extra information.

Step 1. Master story

First of all you need to invent a story. Thats all. You don't need to divide it in parts, create motivations, add extra details. Just invent story how it happens. This story is a master story.

Example

Ring keeper

Gandalf comes back from a long journey to Frodo and tell him about the One Ring and about black riders. Sam overhear Gandalf's story. Gandalf catches Sam and sends Sam and Frodo to Bree. On the way they meet Merry and Pippin and take them along.

Step 2. Master story processing

Lets begin work at master story.

  • Make story character list.

    Gandalf, Frodo, Sam, Merry, Pippin

  • Split story by events where each event happens in one place, one time and with same characters. Then give event a name.

    1. Starting journey

    Gandalf comes back from a long journey to Frodo and tell him about the One Ring and about black riders. Sam overhear Gandalf's story. Gandalf catches Sam and sends Sam and Frodo to Bree.

    2. Merry and Pippin

    On the way Frodo and Sam meet Merry and Pippin and take them along.

Step 3. Adaptations

Before this step we make a master vision of story. At that step we describe event subjective vision for each event participant.

  • 1. Starting journey - Sam

    I pollarded bushes and suddenly hear voices of Frodo and Gandalf (…)

  • 1. Starting journey - Frodo

    Gandalf comes back from a long journey and he was very anxious. (…)

Step 4. Making handouts

On making handouts we gather together all adaptations for each character and make final proof reading.

Sam

Story: Ring keeper

  • 1. Starting journey

    I pollarded bushes and suddenly hear voices of Frodo and Gandalf (…)

  • 2. Merry and Pippin

    We were on acres when suddenly we hear rackets and see two loons (…)

SMTK NIMS workflow

Resume

  • Goal is reached. Handouts are ready.

  • SMTK NIMS is based on described approach. Handouts preparing process is splitted by simple steps and has handy presentation.

  • Moreover there are several secondary LARP tasks and NIMS has tools for them. You can get more information in presentation second part.

Extra tools

Inventory Lists

  • Inventory List - list of items which character should have on game start. First of all these items are prepared by masters.
  • Inventory is related to story. In this case we know where item come from and it is useful in switching character case.
  • Example
    • In story "Starting journey" Frodo gets Sting and mail shirt from Bilbo.
    • In story "Fellowship of the Ring" Frodo gets Galadriel's phial.
  • If some inventory is not related to story you can enumerate it in profile.

Game event planning

  • Game event planning - information for master helpers about what and where they need to do something.
  • For example in some story helper must bring letter on second game night. Or it is necessary to be a monster each midnight.
  • You print helpers handouts before game as usual handouts. You can prepare several types of handouts for several helpers.

Event Timeline

  • You can set event time for each event. This is not necessary but it is recommended. If you set event time then you can print events in chronological order in handouts.
  • If event time is not specified then by default event happens on the game start time.
  • You can see event correlation between different stories in timeline.

Character activity workload

  • Each character has different activity type in each story.
  • We selected 4 activity types:
    • Active - character is active in story. Examples: Frodo, Don Quixote.
    • Follower - character follows somebody but the main target is not important for him. Examples: Sam, Sancho Panza.
    • Defensive - character is running from chasers. Examples: spy in raid, murderer running from police.
    • Passive - character with some information or artifact but without any goals.
  • Character can have several activities simultaneously in one story.
  • Graphical and statistical activity view will help to analyze character workload for the LARP game.

Social bonds

  • We use simple logic based approach. If two characters participate in one event we assume this characters have social interaction which formed social bond.
  • It doesn't matter either characters have each other's back or one character accidently bemired other.
  • The first and the second made a social bond.
  • By collected social bonds SMTK NIMS can draw several social network types:
    • Social bonds - character social bond network.
    • Character-presence-story - network of character-story relations.
    • Character-activity-story - network of character-story relations based on activity type.

Profile filter

  • If you have many characters you need to search appropriate characters for events by some criteria.
  • For example you need to add young single noble to love story.
  • You can filter characters by profile:
    • Age - greater 18
    • Social category - noble
    • Condition - single
  • And sort character list in ascending order

Statistics and Diagrams

  • SMTK NIMS calculates next statistics automatically: quantity of stories/characters/events/symbols and general story completeness.
  • NIMS builds histograms of story event number, story character number and detailed event completeness.
  • NIMS builds automatically histograms for profile number items and donut diagrams for single selection and checkbox items.

Server mode

  • Initially SMTK NIMS was made for offline. But for making big games we need several story masters coordination.
  • For this case we added server mode. There is a common base in the server and several users can work on one game simultaneously (server provide an Internet site in some case).
  • There is a strict rights distinction between server users.
  • Server mode is in beta testing now. We have more then 10 servers right now. If you need server for your game tell us and we make one for you.
